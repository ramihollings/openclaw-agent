Initialize Console

Access to the mr.technology Autonomous Infrastructure Hub is gated via Docker. To initialize your local environment or deploy to production, follow the protocol below.

1. Pull the Core Image

The core engine is distributed via our private registry. Ensure your Docker daemon is running.

docker pull mrtechnology/ecosystem-core:v4.2

2. Configure Environment

Create a secrets.env file in your deployment directory. If you are a free tier user, leave the license key blank.

# secrets.env
MR_TECH_LICENSE_KEY=FOUNDER-YOUR-KEY-HERE
GEMINI_API_KEY=sk-or-v1-...
STRIPE_SECRET_KEY=sk_live_...
PORT=8000

3. Execute Boot Sequence

Run the container, mounting your environment variables.

docker run -d \
  --name mrtech-hub \
  --env-file secrets.env \
  -p 8000:8000 \
  mrtechnology/ecosystem-core:v4.2
