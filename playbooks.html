Technical directives for securing the agent development lifecycle.

Boundary Pining: Restricting Host Access

The most critical step in agent safety is defining the trust boundary. You must explicitly pin your exec allowlists.

# config.yaml example
security:
  allowlist:
    - ls
    - grep
    - git
  sandbox: require

By forcing sandbox: require, you ensure that any tool call that hasn't been mathematically verified is rejected at the gateway.
